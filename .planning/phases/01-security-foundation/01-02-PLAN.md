  .requestMatchers("/api/v1/admin/**").hasRole("ADMIN")

The order matters — more specific matchers must come before .anyRequest().authenticated(). The final authorizeHttpRequests block should be:

  .requestMatchers(HttpMethod.OPTIONS, "/**").permitAll()
  .requestMatchers("/api/v1/health").permitAll()

Step 3 — Create RateLimitFilter at src/api/src/main/java/com/nerdquiz/security/RateLimitFilter.java.

Annotate with @Component. Extend OncePerRequestFilter.

Per spike findings, implement:
- ConcurrentHashMap<String, RequestCounter> for per-IP tracking
- RequestCounter inner class: AtomicLong count, long windowStart
- tryAcquire(String clientIp): uses ConcurrentHashMap.compute() for atomic create-or-reset (thread-safe, no explicit locks)
- maxRequests and windowMillis configurable via @Value("${security.rate-limit.max-requests:60}") and @Value("${security.rate-limit.window-seconds:60}")
- getClientIp(HttpServletRequest): extract from X-Forwarded-For header (first comma-separated value), fallback to getRemoteAddr()
- Override shouldNotFilter: return true for /api/v1/health (bypass rate limiting)
- On rejection: set response status 429, Content-Type application/problem+json, write JSON body with type="urn:nerdquiz:rate-limit-exceeded", title="Rate Limit Exceeded", status=429, detail with retry seconds. Set Retry-After, X-RateLimit-Limit, X-RateLimit-Remaining=0 headers.

Do NOT add a scheduled cleanup task for expired entries — not needed for MVP with <50 users. The map stays bounded by unique IPs.

Step 4 — Create SecurityHeadersConfig at src/api/src/main/java/com/nerdquiz/security/SecurityHeadersConfig.java.

Annotate with @Configuration. Define a @Bean method returning SecurityFilterChain.

Per spike findings, configure http.headers() with:
- contentSecurityPolicy: "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self' https://fonts.gstatic.com; connect-src 'self' https://*.supabase.co wss://*.supabase.co; frame-src 'none'; object-src 'none'; base-uri 'self'; form-action 'self'; upgrade-insecure-requests"
- frameOptions: deny()
- contentTypeOptions: withDefaults() (adds nosniff)
- httpStrictTransportSecurity: maxAgeInSeconds(31536000), includeSubdomains(true)
- referrerPolicy: STRICT_ORIGIN_WHEN_CROSS_ORIGIN
- permissionsPolicy: "camera=(), microphone=(), geolocation=()"

IMPORTANT: In Spring Boot, multiple SecurityFilterChain beans are merged by the framework. The headers configured here will be ADDED to the ones in SecurityConfig (not replaced). However, if both chains configure the same header (e.g., both configure CSP), the last-registered bean wins. To avoid conflicts, do NOT configure headers in SecurityConfig — leave header configuration ONLY in SecurityHeadersConfig.
